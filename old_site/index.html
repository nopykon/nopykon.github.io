SOFTWARE RENDERING PROJECT
(YET ANOTHER)

This engine is written in C, and it fills in polygons using the CPU only, like old DOS games such as Quake or Doom ( except I have an i7, not a 486 ). It uses a 16-bit framebuffer (rather than 256-colors) and no z-buffer. Only integers (fixed point) are used for graphics as well as physics calculations. This style is very close to how old Playstation-games were rendered. Unlike the PS1, perspective correction is performed at the start and end of each row of pixels within a triangle, and if needed, at every 16th pixel within the row. (In the very last step, OpenGL is used to upscale the framebuffer.) The animations are interpolated over fixed physics time steps, untying the framerate from the simulation while still looking smooth. Cel-shading hack
